# __APP_NAME__

A Skal app, generated by `scripts/new-app.sh`.

## Layout

```
__APP_NAME__/
├── package.json         # workspace pkg — pulls in `skal` from packages/skal-js
├── src/
│   ├── App.jsx          # the dev writes this
│   ├── index.jsx        # native entry — `render(<App />, root)`
│   └── web-main.jsx     # web entry — DOM `render(<App />, #app)`
├── vite.config.js       # native bundle (IIFE → JSC eval)
├── vite.config.web.js   # web bundle (ES module → DOM)
├── index.html           # web shell
├── flutter-host/
│   ├── lib/main.dart    # Flutter app entry — boots Skal, mounts <SkalRoot />
│   ├── pubspec.yaml     # path-dep on packages/skal_flutter
│   └── assets/          # populated by `bun run build`
└── README.md            # this file
```

## Workflow

```bash
# Install JS deps (run once from the monorepo root)
cd ../..
bun install

# JS bundle → flutter-host/assets/
cd examples/__APP_NAME__
bun run build

# Add Flutter platform configs the first time. Use --org to set your
# reverse-DNS prefix (becomes the Android package name + iOS bundle id).
cd flutter-host
flutter create --org com.example --project-name __APP_NAME_SNAKE__ --platforms macos,ios,android .

# Install libskal binaries into the new platform configs:
cd ../../..
SKAL_FLUTTER_FRAMEWORKS=$PWD/examples/__APP_NAME__/flutter-host/macos/Frameworks ./scripts/link-libskal-flutter-mac.sh
SKAL_FLUTTER_NATIVE_LIBS=$PWD/examples/__APP_NAME__/flutter-host/android/app/src/main/jniLibs/arm64-v8a ./scripts/link-libskal-flutter.sh

# Run it
cd examples/__APP_NAME__/flutter-host
flutter pub get
flutter run -d macos
```

## Web preview

```bash
# In examples/__APP_NAME__/
bun run dev:web
# → opens http://localhost:5173
```

(Note: the bridge isn't available on web; `skal/renderer-web` renders
to DOM as a degraded preview. Use it for layout iteration, not for
testing Flutter-specific behaviors.)
