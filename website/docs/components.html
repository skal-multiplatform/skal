Components

Fifty-plus fast-path widgets, styled with props, animated by the host, navigable out of the box. Capitalized JSX tags compile to binary ops — they cost nothing at runtime.

Styling is props

<Box
  width={220} height="wrap"
  padding={16} cornerRadius={14}
  background="#FF131A26" borderWidth={1} borderColor="#FF1F2937"
  onTap={() => open()}
/>

Every styling prop maps to a compact binary prop-op with a stable id — there's no stylesheet engine to re-implement and nothing to parse at runtime. Colors are #AARRGGBB; sizes are dp numbers or the "fill"/"wrap" sentinels.

Animation: declare it, the host runs it

// tween any hot prop — runs entirely on the Flutter side
<Box scaleX={pressed() ? 0.96 : 1} animate={{ duration: 180, curve: 'easeOut' }} />

// or real spring physics
<Box translationY={open() ? 0 : 320} spring="bouncy" />

Once the target value crosses the bridge (one op), the host owns the motion — tweens, springs, repeat loops, and drag-release physics all animate at 60 fps with zero per-frame bridge traffic.

Navigation

import { createRouter } from 'skal/runtime';

const router = createRouter({
  home:    { component: () => <Home /> },
  detail:  { component: (p) => <Detail id={p.id} />, title: 'Detail' },
}, 'home');

// push with params; native back gestures + transitions included
router.push('detail', { id: 42 });

Screens stay alive on the stack (scroll positions survive back-navigation), Tabs keep every tab mounted, and Hero transitions work across screens. Dialogs, snackbars, and pickers are one-line imperative calls from skal/renderer.

Accessibility & testability

<Text label="Welcome" testID="home-title" />
<Button label="Save" semanticLabel="Save document" />

semanticLabel feeds screen readers; testID becomes the platform accessibility identifier that E2E tools like Maestro match on. Both ride Flutter's real semantics tree.
